// 01 — PITCH

The hard way, on purpose.

Most game-playing AI plugs straight into the game's code. TrueSkate-AI doesn't get that luxury — it plays True Skate on a real iPhone, sends the same swipes your thumb would, and reads the screen to find out whether a trick landed. A shaped reward turns those readings into a learning signal, and an evolution strategy slowly discovers the gesture that nails each trick. The goal isn't a high score — it's an agent that runs whole park lines in my own skating style.

My conjecture: I can't assimilate the entire internet's knowledge in a lifetime, and I can't train Claude Opus on consumer hardware — but I've proven I can master True Skate well within one. So it may be possible to train an AI to play True Skate on consumer-grade hardware.

And the goal isn't only software. The agent's actuator is a tendon-driven robotic hand being built to physically tap and swipe the glass — moving the way a human hand does, not a script poking at an API.

  • 36 flatground tricks
  • 3 devices in parallel
  • real-time only
// THE HAND

The hand.

Almost every robotic hand is built to grip. TrueSkate-AI's needs the opposite — speed and human-like motion under no load — so it borrows from several open-source designs and recombines them.

I'm experimenting with existing open-source hands — ORCA, HRI, and Ruka-v2 — and folding the best-suited features of each into my own.

Each finger gets a tendon-driven curl plus base splay — roughly two to three degrees of freedom — so it can flick and spread, with capacitive fingertips gated electronically so every tap registers. The first build is a two-finger scaffold (index + middle) to prove the module and the control loop before scaling to all five.

  • ~2–3 DOF per finger
  • tendon-driven
  • zero load-bearing
  • forked from open-source hands
references: ORCA Hand HRI Ruka-v2

// 03 — HOW IT WORKS

How it works.

I started by teaching the agent flatground tricks with CMA-ES — an evolutionary optimizer rewarded by the trick's name shown on-screen, read by OCR and matched against a known tricks table (it landed the 360 flips in the run below). The system is now built around two-stage behavioral cloning: it learns gestures directly from expert play instead of searching for them. Here's the pipeline.

  1. 01

    The demonstrations. Expert True Skate gameplay, captured on real iPhones, is the data the policy learns from — no simulator and no access to the game's code, just what's on the screen.

  2. 02

    The trace extractor. A small U-Net (Model 1) reads each frame and finds the finger on the glass. It's self-labeled from the agent's own gestures — when the agent makes a known swipe, the resulting on-screen trace teaches the network where touch lands — and it recovers the touch point on unseen expert frames to within a few percent.

  3. 03

    The strokes. Per-frame touch points are fit into discrete curved-drag strokes — waypoints, duration, easing and curvature — normalized across three screen sizes so one representation runs on any device in the fleet.

  4. 04

    The policy. A transformer (Model 2) is trained to take the recent frames plus the last few strokes and predict the next stroke — behavioral cloning of the expert, one decision at a time. (An earlier CMA-ES baseline and a collapsed PPO + HER attempt taught me what this noisy, real-time setting could and couldn't give back.)

  5. 05

    The hands. Predicted strokes are executed on real iPhones via Appium + WebDriverAgent — three phones in parallel on a headless server, monitored remotely. The endgame swaps that software touch for a physical, tendon-driven robotic hand (in build) performing the same gestures on the glass.

// 05 — ROADMAP

Roadmap.

Two tracks — the agent and the hand — converging on a piece that plays, and is built to be seen.

Track A · the agent — software

  1. A1 [ now ]

    Per-trick mastery

    Reliable, repeatable landings for individual tricks across the 36-trick flatground set.

  2. A2

    One policy, every trick

    A single skill-conditioned network that performs any trick on command, distilled from the evolution-strategy rollouts.

  3. A3

    Style imitation

    An agent that strings tricks into full park lines that look like how I actually skate.

Track B · the hand — hardware

  1. B1 [ now ]

    Two-finger module

    Curl + splay on index and middle — proving the mechanism and the control loop before scaling up.

  2. B2

    Full five-finger hand

    Scaling the curl-and-splay module to a complete tendon-driven hand.

  3. DISPLAY / GOAL
    B3 [ the goal ]

    Physical play, on a podium

    A left hand holding the iPhone and a right hand finger-skating True Skate on a real screen — finished cleanly enough to mount on a podium and leave on long-term display.
